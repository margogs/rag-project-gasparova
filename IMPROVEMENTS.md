# IMPROVEMENTS

Направления улучшения MVP. Каждый пункт вырос из наблюдаемого поведения на реальных
данных DSLD (1500 шотов, 4943 чанка), а не придуман абстрактно.

## 1. Семантический поиск вместо TF-IDF

**Проблема (наблюдаемая).** На запрос «magnesium shot for stress and sleep» система
выдаёт «Super Stress With Vitamin C», потому что TF-IDF матчит слово *stress*, а не
понимает, что ключевое — *magnesium*. Поиск идёт по словам, не по смыслу.

**План внедрения.** Заменить/дополнить `app/retriever.py` эмбеддингами
(`sentence-transformers`, напр. `all-MiniLM-L6-v2`): кодировать чанки и запрос в векторы,
искать по косинусу. TF-IDF оставить как быстрый baseline и для сравнения качества.

## 2. Фильтр по форме продукта (только настоящие «шоты»)

**Проблема (наблюдаемая).** В выдачу попадают Capsule / Tablet / Softgel
(напр. «Vitamin K2», «Shot-of-Energy» в форме Tablet), хотя проект про жидкие шоты.

**План внедрения.** При сборе данных (`fetch_data.py`) или в `ingest.py` фильтровать по
`physicalState` = Liquid, либо хранить форму в метаданных чанка и отсекать не-Liquid на
этапе поиска. Поле уже есть в данных DSLD.

## 3. Русскоязычные запросы

**Проблема.** Данные DSLD на английском, TF-IDF матчит по словам, поэтому запрос
«шот для сна» по английским этикеткам ничего не находит. Для бота «микс ит» нужен
русский ввод.

**План внедрения.** Добавить шаг перевода запроса (en↔ru) перед поиском, либо при сборе
данных генерировать русские теги-ключевики к каждому продукту и индексировать их вместе
с английским текстом.

## 4. Сопоставление с реальным ассортиментом «микс ит»

**Проблема.** DSLD — это generic-база этикеток США; реального каталога «микс ит» там нет.

**План внедрения.** Завести отдельный небольшой источник с позициями «микс ит» и
сопоставлять найденные в DSLD аналоги с конкретными продуктами автомата (по ключевым
ингредиентам). Это шаг к продуктовому сценарию из брифа.

## Что уже реализовано из улучшений

- **Дедупликация при сборе** (`fetch_data.py`): повторные `_id` со страниц поиска не
  попадают в датасет дважды (множество `seen`).
- **Порог отказа** (`app/config.py: SCORE_THRESHOLD`): при низком score система честно
  отказывает, а не выдаёт нерелевантные шоты как ответ.
