# Conventions

Единый стандарт разработки, чтобы стиль и scope не разъезжались по итерациям.

## Границы изменений

- **Минимальный diff:** меняем только то, что нужно для текущей итерации.
- **KISS:** простое решение предпочтительнее сложного. Никаких эмбеддингов, векторных БД,
  внешних LLM в MVP — только TF-IDF и demo-ответ.
- Новые зависимости — только через `uv add` и с явной причиной.

## Архитектура модулей

Повторяет структуру образца, ответственность не размывается:

| Модуль | Ответственность |
|--------|-----------------|
| `scripts/fetch_data.py` | сбор данных из DSLD API → `data/raw/datasets.json` |
| `scripts/ingest.py` | `datasets.json` → нормализованные `documents.jsonl` |
| `app/chunker.py` | нарезка документов на чанки |
| `scripts/build_index.py` | ingest + chunk + обучение TF-IDF, сохранение индекса |
| `app/retriever.py` | TF-IDF + косинус, top-k |
| `app/generator.py` | сборка demo-ответа из чанков |
| `app/prompts.py` | правила ответа и тексты отказов |
| `app/main.py` | Streamlit UI |
| `app/config.py` | пути, `top_k`, размер чанка, порог отказа |

## Правила ответа

- Отвечаем **только по найденному контексту** (чанкам). Ничего вне данных.
- Всегда показываем источник: `doc_id`, score, фрагмент текста.
- Если top-score ниже порога или данных нет — **отказ** без выдумок.
- **Не выдумываем** дозировки, эффекты, бренды, которых нет в этикетке.
- Запрещены медицинские заявления формата «лечит», «снижает болезнь»
  (требование из брифа «микс ит»); язык — нейтральный, по данным этикетки.

## Тестовые требования

- Минимум 3 теста green (на «хорошо» — 5+):
  - чанкинг режет документ ожидаемым образом;
  - retriever возвращает релевантный чанк по «положительному» запросу;
  - на запрос вне данных срабатывает отказ.
- Тесты запускаются через `uv run pytest tests/ -v`.

## Правила по данным и зависимостям

- Сырые данные — только в `data/raw/datasets.json`; индексируется текст, не CSV.
- Источник, лицензия (CC0) и схема записи фиксируются в `doc/DATA.md`.
- Производные файлы (`data/processed/`, `data/index/`) — генерируются, в git не коммитятся
  (через `.gitignore`).
- Зависимости и версии — в `pyproject.toml` / `uv.lock`.
